## üìà Retention Engineering

### The 100%+ Goal

YouTube rewards videos watched multiple times:

#### Progressive Revelation
- Don't give away everything at start
- Build towards a payoff
- Create "I need to see how this ends" tension

#### Information Gaps
- Open a loop at the start
- Close it only at the end
- Make viewers NEED to complete

#### Loop Design
- End frames that connect to beginning
- "Wait, go back and watch that again"
- Revelation that recontextualizes the start

## üìä Content Lifecycle

### The Viewed vs Swiped Metric

Unique to Shorts:
- "Viewed" = watched more than swipe gesture
- "Swiped away" = negative signal
- Target: 70%+ viewed rate

### Shorts Distribution Phases

```
Phase 1 (0-2h):    Test with subscribers + small batch
Phase 2 (2-24h):   Expansion based on viewed rate
Phase 3 (1-7d):    Continued reach or decline
Phase 4 (7-28d):   Content lifecycle (fresher = better)
```

### 28-Day Freshness

YouTube prefers recent content:
- Old Shorts get less push
- Repurposed TikToks age out
- Create native Shorts content
